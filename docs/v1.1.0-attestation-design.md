# MKPE v1.1.0 Attestation Design

## Purpose

Layer 3 adds signed build attestations to MKPE without changing the `.mkpe` archive format. A `.mkpe` sidecar proves artifact bytes; an attestation proves how a subject artifact or sidecar was produced, when it was observed, and which signing identity made that claim.

The v1.1.0 milestone delivers a shippable JSON attestation workflow through the existing `mkpe` CLI:

```text
artifact -> dna create -> .mkpe sidecar -> attest generate -> build_attestation.json -> attest verify
```

## Non-Goals

- Do not modify the `.mkpe` binary format.
- Do not implement hardware attestation, TPM, SGX, or remote witnesses.
- Do not claim that dependencies, compilers, or machines were uncompromised.
- Do not implement embedded DNA tags in this milestone.

## Attestation Document

`build_attestation.json` is signed JSON. The signature covers every public field except `signature`.

Required fields:

- `schema_version`: attestation schema version, initially `1.0`.
- `attestation_id`: UUID for this attestation event.
- `subject_path`: path string captured at generation time.
- `subject_kind`: `file` or `directory`.
- `subject_sha256`: SHA-256 hash of the attested subject. Directories use a deterministic hash over relative file paths and file hashes.
- `bundle_manifest_id`: optional linked `.mkpe` manifest ID.
- `bundle_root_hash`: optional linked `.mkpe` root hash.
- `build_fingerprint`: OS/user/host/process/MKPE version/runtime context.
- `command`: optional build command metadata.
- `timestamp_utc`: signing timestamp.
- `attested_by`: operator or build-system identity.
- `signer_public_key`: Ed25519 public key that must verify the signature.
- `signature`: Ed25519 signature over the canonical payload.

## CLI

New commands:

```powershell
mkpe attest generate <subject> --key .\keys\mkpe_private.key --output .\build_attestation.json
mkpe attest verify .\build_attestation.json --subject <subject> --public-key .\keys\mkpe_public.key
mkpe attest inspect .\build_attestation.json
```

Optional generation inputs:

- `--bundle <path>` links the attestation to an existing `.mkpe` sidecar.
- `--attested-by <id>` records the operator or build identity.
- `--command <command>` records the build command as text.
- Global `--format json` emits machine-readable success/failure payloads for automation.

Exit codes:

- `0`: attestation verified.
- `2`: trust, signature, or subject hash verification failed.
- `3`: operational/input error such as unreadable files or malformed JSON.

## Verification

Verification must fail when:

- The attestation signature is invalid.
- The trusted public key does not match `signer_public_key`.
- The current subject hash differs from `subject_sha256`.
- The linked `.mkpe` sidecar cannot be loaded or its manifest/root hash differs from the attestation.
- Required fields are malformed or missing.

## Testing

Core tests cover signing, successful verification, tamper detection, wrong-key rejection, subject hash mismatch, and deterministic directory hashing.

CLI tests or smoke checks cover JSON output, missing public key errors, and exit-code behavior.

## Release

The milestone ships as a focused v1.1.0 PR with docs and demo commands. After CI passes, merge to `main` and tag `v1.1.0`.
